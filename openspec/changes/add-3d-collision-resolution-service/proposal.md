## Why

当前未归档变更把障碍物 SDF 描述为“基于局部 AABB 的近似缓存与查询”，并允许在缺少 `coacd` 或 `python-fcl` 时退化为单凸块和 AABB 近似实现。这与项目目标冲突：需求要的是高精度 SDF、精确碰撞真值和严格可追溯的几何求解链路，而不是为了兼容缺依赖环境而保留降级路径。

如果继续沿用现有定义，后续实现、测试和 README 都会围绕近似方案展开，导致服务边界从一开始就偏离目标。因此需要在现有 change 内直接修正方案，明确高精度体素化网格 SDF 与强制原生几何依赖是硬约束，并以此重写实现任务与验收标准。

## What Changes

- 将预处理能力收紧为：必须基于原始三角网格生成高精度体素化网格 SDF 缓存，并记录体素尺寸、包围盒、符号判定方式和缓存来源信息；不再接受局部 AABB/凸块盒并集近似。
- 将 `coacd`、`python-fcl` 以及高精度 SDF 构建所需依赖定义为正式运行前提；缺失依赖时系统必须直接失败并返回明确诊断，而不是切换到近似实现。
- 将碰撞检测与复检能力收紧为仅允许精确路径：`AABB broadphase + CoACD + FCL narrowphase`，移除单凸块回退、AABB 碰撞回退及相关文档语义。
- 将自动修正能力收紧为仅允许查询高精度体素化网格 SDF，并在结果中保留 SDF 分辨率、缓存键和求解参数，确保求解过程可复现。
- 更新任务、验收标准与项目文档，使 README、OpenSpec 与实现目标对齐，不再出现“兼容缺依赖环境”或“近似 SDF 可接受”的表述。

## Capabilities

### New Capabilities

- `asset-preprocessing`: 预处理 3D 网格资源，生成 AABB、CoACD 凸分解、FCL 几何数据、高精度体素化网格 SDF 与表面采样等可复用工件。
- `collision-detection`: 对输入网格执行候选碰撞对过滤与精确碰撞检测，并返回结构化接触结果与诊断信息。
- `collision-resolution`: 对处于碰撞中的可动物体执行基于高精度体素化网格 SDF 的位姿优化，输出候选位姿与求解摘要。
- `collision-verification`: 对候选位姿执行与常规检测一致的精确复检，并给出最终权威结论。

### Modified Capabilities

- 无。

## Impact

- 需要替换当前基于局部 AABB 的 SDF 表达，改为面向原始三角网格的高精度体素化 SDF 预处理与查询链路。
- 需要移除当前代码和文档中的回退实现、可选原生依赖语义以及相关配置开关。
- 需要补充针对依赖自检、网格可处理性、高精度 SDF 缓存与精确复检的一组新测试。
- 需要同步修订 README、安装说明与 OpenSpec 文档，明确项目只支持满足前置依赖的正式环境。
