## Context

项目目标是提供一个基于 Python 的后端服务，用于检测 3D 网格之间的碰撞，并通过尽量小的位姿调整解除碰撞。现有未归档方案已经建立了四段式链路：预处理、精确检测、自动修正、精确复检；但其中两项关键定义不符合目标要求：

- 障碍物 SDF 被定义成基于凸块局部 AABB 的近似距离场，而不是基于原始三角网格构建的高精度体素化网格 SDF。
- `coacd` 与 `python-fcl` 被定义成可缺省依赖，缺失时允许回退到单凸块或 AABB 近似实现。

这会直接破坏方案的可信度边界。高精度修正依赖的不是“能运行的近似场”，而是可控制误差、可缓存、可追溯的网格 SDF；精确检测依赖的也不是“接口兼容的替代物”，而是正式的 CoACD 与 FCL 几何后端。本次设计因此要做的不是继续扩展近似链路，而是在同一 change 内把约束收紧到符合目标的正式方案。

## Goals / Non-Goals

**Goals:**

- 定义基于原始三角网格的高精度体素化网格 SDF 预处理与缓存方案。
- 将 `coacd`、`python-fcl` 和高精度 SDF 构建链路定义为正式运行前提，并在缺失时直接失败。
- 保持 `AABB broadphase + CoACD + FCL narrowphase + SDF 优化 + FCL 复检` 的主流程，但移除所有近似回退路径。
- 让自动修正阶段只消费高精度体素化网格 SDF，并在结果中暴露足够的求解与缓存元数据。
- 让 README、OpenSpec 和后续实现任务对齐为同一套“高精度、无降级”的交付标准。

**Non-Goals:**

- 不提供缺依赖时的兼容模式、降级模式、兜底实现或开发测试专用近似路径。
- 不通过局部 AABB 并集、凸块盒近似或其他盒体距离场替代正式 SDF。
- 不在本次设计中覆盖实时物理仿真、连续碰撞检测、多可动物体联合优化或网格自动修复。
- 不将优化阶段的 SDF 值直接视为最终碰撞真值，最终结论仍由精确复检给出。

## Decisions

### Decision: 保留双轨架构，但两条轨都必须使用正式几何基础

系统仍然采用四个能力模块：

1. `asset-preprocessing`
2. `collision-detection`
3. `collision-resolution`
4. `collision-verification`

在线主流程如下：

```text
mesh/pose input
    |
    v
asset-preprocessing (cache miss 时执行)
    |
    v
AABB broadphase
    |
    v
CoACD part pairs + FCL narrowphase
    |
    +--> no collision -> 返回检测结果
    |
    +--> collision -> volumetric mesh SDF pose optimization
                         |
                         v
                    FCL verification
                         |
                         v
                 返回修正结果与最终结论
```

这里的关键修订不是流程结构，而是“每一步的几何基础必须是正式实现”：

- CoACD 用于非凸网格的正式凸分解。
- FCL 用于 narrowphase 和最终复检。
- SDF 必须来自原始障碍物三角网格的高精度体素化结果，而不是局部盒体并集。
- 任一关键依赖不可用时，流程直接失败，不切换到替代算法。

选择理由：

- 项目目标是高精度碰撞真值与高精度修正，而不是保留一个“弱化但能跑”的备用通路。
- 只有把三条几何链路都绑定到正式实现上，检测、优化和复检之间的误差边界才是可说明的。

### Decision: SDF 必须基于原始三角网格生成高精度体素化距离场

障碍物 SDF 的构建对象必须是标准化后的原始三角网格，而不是凸分解后的局部 AABB。预处理阶段生成的 SDF 工件至少包含：

- 体素网格原点 `origin`
- 体素间距 `spacing`
- 网格尺寸 `shape`
- 有符号距离体 `values`
- 生成该体数据所用的包围盒与 padding 参数
- 体素精度参数，例如目标体素边长或等价分辨率配置
- 用于判定内外符号的算法标识

生成流程要求如下：

1. 对障碍物网格执行严格几何校验，至少确认其满足构建有符号距离场所需的闭合性、可解释法向与非退化三角面要求。
2. 基于障碍物 mesh 的整体包围盒生成规则体素网格，并显式记录 padding。
3. 对每个体素中心计算相对于原始三角网格的有符号距离，形成完整距离体。
4. 在线查询阶段仅允许对该体素 SDF 执行三线性插值或等价连续查询，不再重新退化为盒体距离。

选择理由：

- 高精度修正需要的是对真实网格边界有稳定逼近的距离场，而不是由若干盒体拼接出的粗糙代理。
- 将 SDF 明确定义为体素化网格工件后，缓存键、精度参数和误差来源都可以进入正式契约。

备选方案：

- 局部 AABB / 凸块盒并集 SDF：实现简单，但几何误差不可接受，不能代表目标方案。
- 在线按查询点即时计算网格距离：可行但成本高，不适合作为高频求解链路的正式基础。

### Decision: 原生几何依赖是强制前提，不再暴露可选语义

`coacd`、`python-fcl` 以及高精度 SDF 构建所需依赖必须被视为正式运行依赖，而不是可选增强项。系统在 CLI 启动、服务初始化或首次构建预处理器时必须执行依赖自检：

- 如果 `coacd` 不可用，预处理直接失败。
- 如果 `python-fcl` 不可用，精确检测与复检直接失败。
- 如果高精度 SDF 构建链路缺依赖，SDF 预处理直接失败。

相关约束：

- `pyproject.toml` 中不再保留“缺失原生依赖仍可运行核心功能”的产品语义。
- 配置层不再保留 `require_native_* = false` 这类允许近似回退的开关。
- 诊断信息只用于说明失败原因，不用于宣布系统自动切换到了某个近似后端。

选择理由：

- 项目要求已经明确不接受兼容性回退；继续保留这类开关只会让实现和文档出现双重语义。
- 依赖失败必须尽早暴露，这比在运行中静默退化更符合高精度服务的工程要求。

### Decision: 精确检测与精确复检共享同一条严格后端路径

碰撞检测与复检都必须使用同一套几何工件和同一条精确后端路径：

- broadphase 只负责候选对裁剪。
- narrowphase 只允许 `CoACD + FCL`。
- 复检复用相同的预处理产物与检测判定标准。

缺少精确工件或后端时，结果必须是 `failed`，而不是“视为无碰撞”或“切换为 AABB 近似”。

选择理由：

- 检测与复检若基于不同精度等级的几何通路，会让“修正成功”的定义失去一致性。
- 统一后端路径可以保证所有碰撞结论都在同一几何基准上产生。

### Decision: 自动修正只消费高精度体素化网格 SDF，并记录完整求解来源

优化阶段的碰撞损失来自可动物体表面采样点对障碍物体素 SDF 的查询。结果中必须保留：

- 所使用的 SDF 缓存键
- SDF 体素精度参数
- 优化器关键参数
- 停止原因、迭代次数、初始/最终损失

自动修正阶段若发现任一障碍物缺少正式 SDF 工件，必须直接失败；不允许构造临时近似 SDF 或退回盒体距离。

选择理由：

- 修正效果是否可信，取决于所使用 SDF 的来源与分辨率是否可追溯。
- 去掉临时近似构造后，优化阶段的误差边界才能稳定地与预处理配置关联。

### Decision: 输入约束要前置失败，而不是事后补救

高精度体素化 SDF 与精确碰撞检测都依赖输入网格满足基本几何前提。本方案要求在预处理阶段前置拒绝以下问题：

- 退化三角形
- 非法索引或非有限数值
- 不满足 SDF 构建前提的障碍物网格
- 无法解释的单位或坐标系

系统不对这类输入执行自动修复、猜测性补全或后处理补救，而是返回明确失败原因。

选择理由：

- 目标是严谨、可复现的几何求解链路。输入前提不成立时直接失败，优于在后续阶段传播不可靠工件。

## Risks / Trade-offs

- [高精度体素化 SDF 的预处理成本显著高于盒体近似] -> 通过缓存复用、离线预处理和显式精度参数控制成本，而不是降低算法精度。
- [高分辨率 SDF 会带来更高内存占用] -> 通过体素边长、包围盒 padding 与缓存生命周期管理约束资源开销。
- [原生依赖安装门槛高于纯 Python 方案] -> 将其明确定义为项目前提，并在安装文档与启动检查中一次性暴露问题。
- [部分网格无法满足高精度 SDF 前提] -> 在预处理阶段直接拒绝，并要求上游提供满足条件的输入资产。

## Migration Plan

1. 收紧依赖契约：将原生几何依赖与高精度 SDF 构建链路提升为正式前提，并删除允许近似回退的配置开关。
2. 重写预处理：基于原始三角网格生成高精度体素化 SDF，并把精度参数纳入缓存键。
3. 重写检测后端选择：移除单凸块与 AABB 回退实现，统一为 `CoACD + FCL`。
4. 重写优化查询：从正式 SDF 工件读取距离场，不再基于局部盒体集合查询。
5. 更新测试与文档：补齐依赖失败、输入拒绝、高精度 SDF 与精确复检的测试，修正 README 与说明文档。
